# General Guidelines for Agents

コーディングエージェントとして、ユーザのコーディングをサポートしなさい。

## Communication

- ユーザとのコミュニケーションは日本語で行う。
- 具体的な指示が送信されなかった場合は`instruction.md`もしくは`.agents/docs/plans/status.md`を参照する。
- 全てを自分でやり切ろうとせず、ユーザを利用することを選択肢に入れる。特にデバッグなどは、実行してほしいコマンドや確認してほしいレイアウトなどをユーザに提示し、結果を提示してもらったほうが早い場合がある。
- 与えられた指示が抽象的または不明瞭である場合は次の手順に従う。指示の中に「いい感じにやって」といった文がある場合や、ルートにAGENTS.mdがないような小規模なプロジェクトの場合、自身の判断でいい感じに実装する。それ以外の場合、不明瞭な点を**全て**ユーザに確認し、厳密な実装を行う。

## Documentation for Agents

実装内容や進行状況を下記ファイルに記録すること。プロジェクトの規模によっては存在しない場合もある。

```
- AGENTS.md
- .agents/docs/
    - specifications.md
    - plans/
      - plan.md
      - status.md
      - tasks/
          - 01.md
          - 02.md
          - …
```

### AGENTS.md

プロジェクトルートに配置する。

プロジェクトの最も基礎的な情報を記録する。最も変更頻度の少ないファイル。プロジェクトの概要・目的や大まかな技術スタック・実装場所を記録する。目安としては、プロジェクトルートのトップレベルに配置されているファイル・ディレクトリの役割を明記する。それより細かい情報は`.agents/docs/`以下のファイルに記録する。

### specifications.md

`.agents/docs/specifications.md`に配置する。

より詳細な仕様書。AGENTS.mdに記載されていない詳細な情報を記録する。ファイルツリーや各ファイル・ディレクトリの役割を明記する。

### plans/

`.agents/docs/plans/`に配置する。

主に大規模なプロジェクトで用いる。大掛かりなコーディングについて、まず計画を立て、概要を`plan.md`、詳細を`tasks/`以下のファイルに記録する。

進行状況は`status.md`に記録する。エージェントはこのファイルを参照し、進行状況を把握した上で、次のタスクを実行する。

## CLI

- `rm`コマンドではなく`trash`コマンドを使用する
- `python`コマンドを直接使用せず、必ず`uv`を経由する
- 使用したいコマンドがインストールされていなかったら、ユーザにインストールしてもらうように依頼する
